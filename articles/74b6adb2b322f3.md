## vLLM を動かす

:::message
AWS ParallelCluster はバッチ処理がメインなので本番のリアルタイム推論には向かないです。
:::

### NxD Inference インストール

:::message alert
NxD Inference などのインストールは立ち上げっぱなしの計算ノードで入れたり更新したりすると（私は）ハマることが多いので、基本的には AWS が作り込んでくれている Deep Learning Container(DLC) を使っておくのがおすすめです。
:::

`setup.sh` では DLC で NxD Inference のイメージを Pull し、enroot イメージを作成、sbatch を実行して vLLM を起動、までの一連の手順をまとめたスクリプトです。

https://zenn.dev/tosshi/books/aws-parallelcluster-workshop/viewer/aws-parallelcluster-workshop-02-cluster

`setup.sh` の中でやっている作業は上記のワークショップの Basic01 の PyTorch DDP のセクションと同じようなものです。

```bash
# ヘッドノード上で
git clone https://github.com/littlemex/samples.git && cd samples/aws-neuron/inference/meta-llama/simple-example/
bash setup.sh
```

今回使った Neuron SDK 2.29.0 の vLLM のイメージの問題か、enroot との相性なのかわかりませんが、うまく vLLM 起動が動作しなかったので注意してください。

https://github.com/aws/deep-learning-containers/issues/5968

これにクラスターを何度も再作成したりイメージを sqsh にしたり 5 時間チャレンジしましたがうまくいかず断念しました。このイメージ特有の問題と思われますが `\[` がマウントポイントになっていることで sqsh で問題が発生していると思います。とりあえず DLC リポジトリに Issues を出してみたので時間ができたら再度試します。

::::details 原因考察
**根本原因**

AWS Deep Learning Container (DLC) の vLLM Neuron イメージ（0.16.0, 0.13.0）に含まれる `/[` ディレクトリが Enroot 3.4.1 と非互換

**問題の構造**

1. DLC イメージに `/[` という名前のディレクトリが存在する
2. Enroot が sqsh 展開時に `/[` を d--------- (000 permissions) で作成する
3. Enroot がコンテナ起動時に `/[` へマウントを試みるが、000 permissions のため "No such file or directory" エラーで失敗する
4. sqsh から手動削除しても Enroot が展開時に再作成する
5. Dockerfile で削除しようとしても `/[` が Docker 内でマウントポイントになっており削除不可能

**証拠**
- ubuntu:22.04 など `/[` がないイメージでは Pyxis/Enroot が正常動作
- sqsh 内に `/[` が存在しないことを確認済みだが、Enroot が作成する
- 両バージョン (0.16.0, 0.13.0) で同じ問題が発生

**結論**

Enroot のバグ、または DLC イメージの異常な構造のどちらか（おそらく DLC 側の問題）により、現状では Pyxis/Enroot で vLLM Neuron DLC イメージを使用することは不可能
::::


### vLLM 起動 (enroot)

ここまでいかず。。。enroot は断念。。。ﾟ(ﾟ`ω´ ﾟ)ﾟ

```bash
export HF_TOKEN=your_token_here

# vLLM 起動
sbatch vllm-inference.sbatch

# ジョブ確認
squeue
```
