# 学習内容と構築するもの

このセクションでは、Transformer を使用して音声を文字に変換する方法について学びます。この技術は音声認識（speech recognition）として知られています。

音声認識は、自動音声認識（Automatic Speech Recognition, ASR）または音声テキスト変換（Speech-to-Text, STT）としても知られており、音声言語処理タスクの中で最も人気があり、エキサイティングなタスクの 1 つです。この技術は、ディクテーション、音声アシスタント、ビデオキャプション、会議の文字起こしなど、幅広いアプリケーションで使用されています。

皆さんはおそらく、気づかないうちに音声認識システムを何度も使用したことがあるでしょう。スマートフォンのデジタルアシスタント（Siri、Google Assistant、Alexa など）を考えてみてください。これらのアシスタントを使用する際、最初に行われることは、音声を文字に文字起こしすることです。その後、文字起こしされたテキストは、天気予報の検索などのダウンストリームタスクに使用されます 🌤️。

以下の音声認識デモを試してみてください。マイクを使用して自分で録音するか、音声サンプルをドラッグアンドドロップして文字起こしすることができます。

音声認識は、音声とテキストの両方に関する知識を必要とするため、困難なタスクです。入力される音声には多くのバックグラウンドノイズが含まれている場合があり、さまざまなアクセントを持つ話者によって話される可能性があるため、音声を正確に認識することが難しくなります。また、書かれたテキストには、句読点のような音響的な音を持たない文字が含まれている場合があり、音声だけから推測することは困難です。これらは全て、効果的な音声認識システムを構築する際に克服しなければならない障壁です。

![音声認識システムの概要図](https://huggingface.co/datasets/huggingface-course/audio-course-images/resolve/main/asr_diagram.png)

タスクを定義したので、音声認識についてさらに詳しく見ていきましょう。このユニットの終わりには、利用可能なさまざまな事前学習済み音声認識モデルと、🤗 Transformers ライブラリでそれらを使用する方法について、しっかりとした基礎知識が身につくでしょう。また、ドメインや言語に応じて ASR モデルをファインチューニングする手順も学び、どのようなタスクに遭遇しても高性能なシステムを構築できるようになります。さらに、音声を文字に変換するライブデモを構築することで、友人や家族にモデルを披露することができるようになります。

具体的には、以下の内容をカバーします。

このユニットでは、音声認識のための事前学習済みモデルについて学びます。まず、利用可能な主要なモデルアーキテクチャとその特徴を理解します。次に、適切なデータセットの選択方法について学びます。データセットの品質とサイズは、モデルの性能に大きな影響を与えるため、タスクに適したデータセットを選ぶことが重要です。

評価とメトリクスについても深く掘り下げます。音声認識システムの性能を正確に測定するためには、適切な評価指標を理解することが不可欠です。Word Error Rate（WER）などの標準的なメトリクスについて学び、それらをどのように解釈するかを理解します。

Trainer API を使用した ASR システムのファインチューニング方法についても実践的に学びます。事前学習済みモデルを特定のドメインや言語に適応させることで、より高精度な音声認識システムを構築できるようになります。ファインチューニングのプロセスでは、データの前処理、モデルの設定、トレーニングパラメータの調整など、実践的なスキルを身につけます。

最後に、構築したモデルをデモとして公開する方法を学びます。これにより、作成した音声認識システムを他の人と共有し、実際に試してもらうことができます。ハンズオン演習を通じて、学んだ知識を実践し、独自の音声認識システムを構築する経験を積むことができます。

これらのトピックを通じて、音声認識技術の全体像を理解し、実際のアプリケーションに応用できる実践的なスキルを習得することができるでしょう。
